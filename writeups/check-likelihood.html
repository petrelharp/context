Computing sequence likelihoods

Sun Nov 15 21:47:09 2015

The basic result is that if \(L_k\) is the log-likelihood of all the \((k,w)\) T-mers in a sequence, and \(w\) and \(k\) are long enough, then \(L_{k+1}-L_k\) should equal the likelihood of the entire sequence.

Lets check this.

Computing the likelihood

Heres a function to compute the likelihood. Note that the full likelihood of the counts includes a combinatorial factor for where the T-mers occur, but we dont want that factor.

# opt will be a list with 'longwin', 'shortwin', and 'leftwin' defined.

likelihood_fun <- function (opt,config,seq) {
    ## count-seq: count T-mers
    longpats <- getpatterns(opt$longwin,config$bases)
    shortpats <- getpatterns(opt$shortwin,config$bases)
    counts <- counttrans.list( list(longpats,shortpats), 
        seqlist=seq[c("initseq","finalseq")],
        leftwin=opt$leftwin, bases=config$bases, 
        shift=0, cyclic=TRUE )

    ## make-genmatrix
    genmatrix <- do.call( makegenmatrix, c( list( patlen=opt$longwin ), boundary="none", config ) )
    projmatrix <- collapsepatmatrix( ipatterns=rownames(genmatrix), leftwin=leftwin(counts), fpatterns=colnames(counts) )
    # this is collapsed transition matrix
    subtransmatrix <- computetransmatrix( genmatrix, projmatrix, tlen=seq$tlen, time="fixed")
    # return log-likelihood (*not* negative log-likelihood)
    ans <- sum( counts@counts * log(subtransmatrix) )
    dummy <- function() { 1 } # this will allow us access back to this environment

    return(list(loglik=ans,dummy=dummy))
}

With a prior on the branch length

Now, lets modify the function to allow for a Gamma prior on the branch length, with shape \(\alpha\) and scale \(\theta\).

# opt will be a list with 'longwin', 'shortwin', and 'leftwin' defined.
likelihood_fun_gamma <- function (opt,config,seq,shape,scale) {
    ## count-seq: count T-mers
    longpats <- getpatterns(opt$longwin,config$bases)
    shortpats <- getpatterns(opt$shortwin,config$bases)
    counts <- counttrans.list( list(longpats,shortpats), 
        seqlist=seq[c("initseq","finalseq")],
        leftwin=opt$leftwin, bases=config$bases, 
        shift=0, cyclic=TRUE )
    ## make-genmatrix
    genmatrix <- do.call( makegenmatrix, c( list( patlen=opt$longwin ), boundary="none", config ) )
    projmatrix <- collapsepatmatrix( ipatterns=rownames(genmatrix), leftwin=leftwin(counts), fpatterns=colnames(counts) )
    # this is collapsed transition matrix
    subtransmatrix <- computetransmatrix( genmatrix, projmatrix, tlen=scale, shape=shape, time="gamma")

    return(list(loglik=ans,dummy=dummy))

The first model

In the first model, in the expression for the likelihood we just need to adjust the probability of having mutated or not; the probability that a Gamma(\(k\),\(\theta\)) occurs before a Exponential(4) is \[\begin{aligned} \int_0^\infty \frac{1}{\Gamma(k)\theta^k} t^{k-1} e^{-t/\theta} e^{-4t} dt &= \int_0^\infty \frac{1}{\Gamma(k)\theta^k} t^{k-1} e^{-t\frac{4\theta+1}{\theta}} dt \\ &= \frac{1}{\Gamma(k)\theta^k} \left(\frac{\theta}{4\theta+1}\right)^k \int_0^\infty u^{k-1} e^{-u} du \\ &= \frac{1}{(4\theta+1)^k} \end{aligned}\] (also the Laplace transform for a Gamma distribution) and so the likelihood of a sequence with \(m\) sites differing is, as above, \[\begin{aligned} \left( \frac{1}{(4+1)} + \left(1-\frac{1}{(4\theta+1)}\right) \frac{1}{4} \right)^{n-m} \left( \left(1-\frac{1}{(4\theta+1)}\right) \frac{1}{4} \right)^m . \end{aligned}\]

Fixed times

Let \(P_t = e^{tG}\). This involves computing \(P_t V\), where \(V\) is a projection matrix. If weve already computed this for \(t\), and want to check the likelihood for \(t+s\), then we need \(P_{t+s} V = P_s P_t V\) so, the previous sub-transition matrix \(P_t V\) can be re-used for larger times.

t_lhoods <- function (opt, config, seq, dt, ndt) {
    ans <- numeric(ndt)
    longpats <- getpatterns(opt$longwin,config$bases)
    shortpats <- getpatterns(opt$shortwin,config$bases)
    counts <- counttrans.list( list(longpats,shortpats), 
        seqlist=seq[c("initseq","finalseq")],
        leftwin=opt$leftwin, bases=config$bases, 
        shift=0, cyclic=TRUE )
    genmatrix <- do.call( makegenmatrix, c( list( patlen=opt$longwin ), boundary="none", config ) )
    projmatrix <- collapsepatmatrix( ipatterns=rownames(genmatrix), leftwin=leftwin(counts), fpatterns=colnames(counts) )
    subtransmatrix <- projmatrix
    for (k in 1:ndt) {
        subtransmatrix <- computetransmatrix( genmatrix, subtransmatrix, tlen=dt, time="fixed")  # add time dt
        # return log-likelihood (*not* negative log-likelihood)
        ans[k] <- sum( counts@counts * log(subtransmatrix) )
    }
    return(ans)
}
lower_opt <- function (x) { x$shortwin <- (x$shortwin-1); x }

Exponential times

Consider the case \(k=1\), i.e., Exponential. Let \(\lambda = \max_i -G_{ii}\) and \(Q_{ij} = G_{ij}/\lambda\) and \(Q_{ii}=-\sum_{j \neq i} G_{ij}\), so that the process jumps as \(Q\) at the times of a Poisson(\(\lambda\)) process. As described in context/writeups/gamma-prob.html, to move from an Exponential(\(1/\theta\)) time to an Exponential(\(1/(\theta+u)\)) time, we need to add either zero jumps (with probability \(\theta/(\theta+u)\)) or a Geometric(\(\lambda(\theta+u)/(1+\lambda(\theta+u))\)) number of jumps. This is done by extend.expAtv; a similar thing could be done for Gamma times with shape not equal to 1.

e_lhoods <- function (opt, config, seq, dscale, ndscale) {
    ans <- numeric(ndscale)
    longpats <- getpatterns(opt$longwin,config$bases)
    shortpats <- getpatterns(opt$shortwin,config$bases)
    counts <- counttrans.list( list(longpats,shortpats), 
        seqlist=seq[c("initseq","finalseq")],
        leftwin=opt$leftwin, bases=config$bases, 
        shift=0, cyclic=TRUE )
    genmatrix <- do.call( makegenmatrix, c( list( patlen=opt$longwin ), boundary="none", config ) )
    projmatrix <- collapsepatmatrix( ipatterns=rownames(genmatrix), leftwin=leftwin(counts), fpatterns=colnames(counts) )
    # subtransmatrix <- computetransmatrix( genmatrix, projmatrix, tlen=dscale, time="gamma")  # add scale dscale
    subtransmatrix <- projmatrix
    scale <- 0
    for (k in 1:ndscale) {
        subtransmatrix <- extend.expAtv( A=genmatrix, scale=scale, dscale=dscale, gAtv=subtransmatrix )  # add scale dscale
        # return log-likelihood (*not* negative log-likelihood)
        ans[k] <- sum( counts@counts * log(subtransmatrix) )
    }
    return(ans)
}
lower_opt <- function (x) { x$shortwin <- (x$shortwin-1); x }

Gamma inference

Let \(f(t;k,\theta)\) be the density function for a Gamma distribution with shape \(k\) and scale \(\theta\) (and hence mean \(k \theta\)). Define \[ P_{k,\theta} = \int_0^\infty f(t;k,\theta) P_t dt ,\] i.e., the transition matrix after a Gamma(\(k\),\(\theta\))-distributed time. If \(T_i\) are independent, Gamma(\(k_i\),\(\theta\)), then \(\sum_i T_i\) is Gamma(\(\sum_i k_i\),\(\theta\)), and so \[\begin{aligned} P_{k_1,\theta} P_{k_2,\theta} &= P_{k_1+k_2,\theta} . \end{aligned}\]

For \(k\) an integer, since a Gamma(\(k\),\(\theta\)) is the sum of \(k\) independent Exponential(\(1/\theta\)) times, and by Poisson thinning, a Gamma(\(k\),\(\theta/(1-\epsilon)\)) is the sum of a NegativeBinomial(\(k\),\(1-\epsilon\)) number of Exponential(\(1/\theta\)) times, i.e., the number of times to flip a \(1-\epsilon\) coin until getting \(k\) heads, so \[\begin{aligned} P_{k,\theta/(1-\epsilon)} &= \sum_{n \ge 0} \binom{n+k-1}{n} \epsilon^{n} (1-\epsilon)^k P_{1,\theta}^{n+k} . \end{aligned}\] Or, to get a Gamma(\(k\),\(\theta/(1-\epsilon)\)), we could start with a Gamma(\(k\),\(\theta\)), and add on \(M\) indepenent Exponential(\(1/\theta\)) times, where \(M\) is NegativeBinomial(\(N\),\(1-\epsilon\)) and \(N\) is Binomial(\(k\),\(\epsilon\)).

Extending gamma times

Now consider the case \(k=1\). By Poisson thinning, a Geometric(\(p\)) number of independent Exponential(\(\lambda\)) times is Exponential(\(\lambda p\)); and therefore \[\begin{aligned} P_{1,\theta/(1-\epsilon)} &= \sum_{n \ge 1} \epsilon^{n-1} (1-\epsilon) P_{1,\theta}^n \\ &= \sum_{n=1}^k \epsilon^{n-1} (1-\epsilon) P_{1,\theta}^n + O(\epsilon^k) . \end{aligned}\]

extend_transmatrix <- function (P,a,da,eps=da/(a+da),tol=1e-8) {
    # if P is e^{TG} where T is Exp(a)
    #   return e^{SG} where S is Exp(a+da), with da>0.
    # as above, a+da = a/(1-eps), so eps=(da/a)/(1+da/a)
    ans <- Pm <- (1-eps)*P
    m <- ceiling(log(tol)/log(eps))
    # cat("m=",m,"\n")
    for (k in seq(2,length.out=(m-1))) {
        Pm <- eps * P %*% Pm
        ans <- ans + Pm
    }
    return(ans)
}
# prob an exponential(theta) is greater than an exponential(1) is 1/(1+theta), so:
stopifnot( abs( 1/(1+0.25/.8) - extend_transmatrix( 1/(1+0.25), eps=0.2 ) ) <= 1e-8 )
stopifnot( abs( 1/(1+0.25/.8) - extend_transmatrix( 1/(1+0.25), eps=0.2, tol=1e-15 ) ) <= 1e-15 )
